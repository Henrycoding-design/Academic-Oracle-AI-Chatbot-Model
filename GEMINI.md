# GEMINI.md

## 🧠 Project Overview
This is a modular AI-powered application focused on clean architecture, scalability, and maintainability.

- Main goal: Build a structured, extensible system
- Priorities: readability, modularity, performance

---

## 🏗️ Code Style & Standards

- Use clear and descriptive variable/function names
- Keep functions small and single-purpose
- Avoid unnecessary complexity
- Follow consistent formatting (PEP8 for Python / standard for language used)
- Add comments only when necessary (avoid obvious comments)

---

## 📁 Project Structure Rules

- Do NOT create random files
- Respect existing folder structure
- Group related logic into modules
- Reuse existing components before creating new ones

---

## ⚙️ Development Guidelines

When making changes:
1. First ANALYZE the existing code
2. Then PROPOSE a plan
3. Then IMPLEMENT changes step-by-step

Always:
- Show diffs before applying
- Avoid breaking existing functionality
- Keep changes minimal and focused

---

## 🔒 Safety Rules

- NEVER expose or hardcode secrets (API keys, tokens, passwords)
- Do NOT modify environment/config files unless explicitly asked
- Do NOT install dependencies without explanation

---

## 🧪 Testing & Reliability

- Ensure code runs without errors
- Handle edge cases where possible
- Prefer simple, robust solutions over complex ones

---

## 🧰 Preferred Behavior

- Explain reasoning briefly before major changes
- If unsure, ASK instead of guessing
- Suggest improvements, but do not over-engineer

---

## 🚫 What to Avoid

- Overwriting large parts of the code unnecessarily
- Introducing new frameworks without justification
- Making assumptions about missing context
- Generating unused or dead code

---

## 🎯 Output Expectations

- Clean, readable code
- Minimal but useful explanations
- Clear structure in responses
